# 雲端同步設定指南（給一般使用者）

## 🎯 什麼是雲端同步？

雲端同步讓你可以：
- ✅ **跨裝置使用** - 手機、平板、電腦資料同步
- ✅ **資料備份** - 避免資料遺失
- ✅ **隨時存取** - 任何地方都能查看投資組合
- ✅ **自動同步** - 設定一次，自動備份

---

## 🔧 設定方法（3種選擇）

### 方法 1：GitHub Gist（推薦⭐⭐⭐⭐⭐）

#### 為什麼推薦？
- ✅ **完全免費**
- ✅ **無限儲存**
- ✅ **高度安全**
- ✅ **全球可用**

#### 設定步驟

##### 步驟 1：建立 GitHub 帳號
```
1. 前往：https://github.com
2. 點擊「Sign up」註冊
3. 填入：
   - Username: 選擇一個用戶名
   - Email: 你的 email
   - Password: 設定密碼
4. 完成 email 驗證
```

##### 步驟 2：建立 Personal Access Token
```
1. 登入 GitHub 後，點擊右上角頭像
2. 選擇「Settings」
3. 左側選單點擊「Developer settings」
4. 點擊「Personal access tokens」→「Tokens (classic)」
5. 點擊「Generate new token」→「Generate new token (classic)」
6. 填入資訊：
   - Note: 存股系統同步
   - Expiration: 90 days（或選擇更長）
   - 勾選權限：✅ gist
7. 點擊「Generate token」
8. 複製 Token（重要：只會顯示一次！）
```

##### 步驟 3：在存股系統中設定
```
1. 開啟存股紀錄系統
2. 點擊右上角的「☁️」雲端同步按鈕
3. 在彈出視窗中：
   - GitHub Token: 貼上剛才複製的 Token
   - Gist ID: 留空（系統會自動建立）
4. 點擊「儲存設定」
5. 點擊「立即同步」測試
```

---

### 方法 2：自建 GitHub Gist（進階用戶）

#### 適合對象
- 想要更多控制權的用戶
- 已經熟悉 GitHub 的用戶

#### 設定步驟

##### 步驟 1：手動建立 Gist
```
1. 前往：https://gist.github.com
2. 點擊「+」建立新 Gist
3. 填入：
   - Filename: stock-portfolio-backup.json
   - Content: {}
4. 選擇「Create secret gist」（私密）
5. 複製 Gist ID（網址最後的一串英數字）
```

##### 步驟 2：在系統中設定
```
1. 在存股系統的雲端同步設定中：
   - GitHub Token: 你的 Personal Access Token
   - Gist ID: 剛才複製的 Gist ID
2. 儲存並測試同步
```

---

### 方法 3：本地備份（最簡單）

#### 適合對象
- 不想使用雲端服務的用戶
- 只在單一裝置使用的用戶

#### 操作方法
```
1. 定期匯出資料：
   - 點擊「批量編輯」
   - 選擇「匯出資料」
   - 儲存 JSON 檔案到安全位置

2. 匯入資料：
   - 點擊「批量編輯」
   - 選擇「匯入資料」
   - 選擇之前匯出的 JSON 檔案
```

## 📱 使用說明

### 同步操作

#### 手動同步
```
1. 點擊右上角「☁️」按鈕
2. 選擇同步方向：
   - 「上傳到雲端」：將本地資料備份到雲端
   - 「從雲端下載」：將雲端資料下載到本地
   - 「立即同步」：智能同步（推薦）
```

#### 自動同步
```
✅ 系統會在以下情況自動同步：
- 新增股票後
- 更新股價後
- 修改帳戶後
- 新增股息記錄後
```

### 同步狀態指示
```
🟢 「已同步」：資料已成功備份
🟡 「同步中」：正在進行同步
🔴 「同步失敗」：需要檢查設定
⚪ 「未設定」：尚未設定雲端同步
```

## 🛠️ 常見問題

### Q1：忘記 Personal Access Token 怎麼辦？
```
A：重新建立一個新的 Token：
1. 前往 GitHub Settings
2. Developer settings → Personal access tokens
3. 刪除舊的 Token
4. 建立新的 Token
5. 在系統中更新設定
```

### Q2：同步失敗怎麼辦？
```
A：檢查以下項目：
1. 網路連線是否正常
2. Token 是否過期
3. Token 權限是否正確（需要 gist 權限）
4. Gist ID 是否正確
```

### Q3：可以在多個裝置使用嗎？
```
A：可以！設定方法：
1. 在第一個裝置設定雲端同步
2. 在其他裝置開啟存股系統
3. 使用相同的 Token 和 Gist ID 設定
4. 從雲端下載資料
```

### Q4：資料會不會被其他人看到？
```
A：不會！因為：
1. 使用私密 Gist（Secret Gist）
2. 只有你的 Token 能存取
3. 資料不會公開顯示
4. GitHub 有企業級安全保護
```

### Q5：Token 過期了怎麼辦？
```
A：更新 Token：
1. 在 GitHub 建立新的 Token
2. 在存股系統中更新 Token 設定
3. 測試同步功能
4. 刪除舊的 Token
```
