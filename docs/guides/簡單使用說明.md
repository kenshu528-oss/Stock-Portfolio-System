# 如何分享經驗到其他 KIRO 專案

## 問題
你花了很多時間修復雲端同步功能，不想在其他專案重複相同錯誤。

## 解決方案
把經驗變成「規則檔案」，讓 KIRO 自動提醒你。

## 最簡單的做法

### 方法 1：手動複製（推薦新手）

1. **找到這個檔案**：`UNIVERSAL_CLOUD_SYNC_STANDARDS.md`

2. **複製到全域位置**：
   - 按 `Win + R`，輸入：`%USERPROFILE%`
   - 創建資料夾：`.kiro\settings\steering`
   - 把檔案複製進去
   - 重新命名為：`cloud-sync-development.md`

3. **完成**！以後所有新的 KIRO 專案都會自動有這個規則。

### 方法 2：使用腳本（推薦熟手）

1. **執行腳本**：雙擊 `setup-global-standards.bat`
2. **按 y 確認**
3. **完成**！

## 效果

以後在任何 KIRO 專案開發雲端同步功能時，KIRO 會自動提醒你：
- 不要依賴本地狀態（像 localStorage）
- 要有完整的錯誤處理
- 要使用統一的資料處理方法
- 要自動切換用戶狀態

## 測試

1. 開啟任何一個 KIRO 專案
2. 開始寫雲端同步相關代碼
3. KIRO 應該會顯示相關的開發建議

## 如果不生效

檢查檔案是否放在正確位置：
```
C:\Users\你的用戶名\.kiro\settings\steering\cloud-sync-development.md
```
