# 替代分享方法指南

## 🎯 目標
為沒有 GitHub 帳號的一般使用者提供多種分享和部署方案。

---

## 🌐 方法 1：免費網頁託管服務

### Netlify Drop（最簡單）
**優點**: 拖拉即可，無需註冊
**步驟**:
```
1. 前往：https://app.netlify.com/drop
2. 將整個專案資料夾拖拉到頁面上
3. 等待上傳完成
4. 獲得免費網址（如：https://amazing-name-123456.netlify.app）
5. 分享網址給其他人使用
```

### Vercel（需註冊，但簡單）
**優點**: 速度快，功能強大
**步驟**:
```
1. 前往：https://vercel.com
2. 註冊帳號（可用 Google/GitHub）
3. 點擊 "New Project"
4. 上傳專案檔案
5. 部署完成後獲得網址
```

### Surge.sh（命令列工具）
**優點**: 快速部署，支援自訂網域
**步驟**:
```bash
# 1. 安裝 Node.js 和 npm
# 2. 安裝 Surge
npm install -g surge

# 3. 在專案目錄執行
surge

# 4. 按照提示設定網域名稱
# 5. 獲得網址（如：your-project.surge.sh）
```

## 📁 方法 2：雲端儲存分享

### Google Drive
**步驟**:
```
1. 將專案打包成 ZIP 檔案
2. 上傳到 Google Drive
3. 右鍵點擊檔案 → "取得連結"
4. 設定為"知道連結的使用者"
5. 分享連結給其他人下載
```

**使用說明文件**:
```
下載後：
1. 解壓縮檔案
2. 雙擊 index.html 開啟
3. 開始使用存股紀錄系統
```

### Dropbox
**步驟**:
```
1. 上傳專案檔案到 Dropbox
2. 建立共享連結
3. 其他人可直接下載使用
```

### OneDrive
**步驟**:
```
1. 上傳到 OneDrive
2. 右鍵 → "共用"
3. 設定權限為"檢視"
4. 複製連結分享
```

## 💾 方法 3：本地分享

### USB 隨身碟
**適用**: 面對面分享
**步驟**:
```
1. 將專案複製到 USB
2. 直接給其他人
3. 對方複製到電腦後雙擊 index.html 使用
```

### 區域網路分享
**適用**: 同一網路環境
**步驟**:
```
1. 在專案目錄開啟命令提示字元
2. 執行：python -m http.server 8000
3. 其他人瀏覽：http://你的IP:8000
4. 直接在瀏覽器使用
```

## 📧 方法 4：電子郵件分享

### 壓縮檔案分享
**步驟**:
```
1. 將專案壓縮成 ZIP 檔案
2. 透過電子郵件寄送
3. 收件人下載解壓縮後使用
```

**注意事項**:
- 檔案大小限制（通常 25MB）
- 可能被防毒軟體攔截
- 建議使用雲端連結代替附件

## 🎁 方法 7：打包分發

### 建立安裝包
**工具**: Electron 或 Tauri
**優點**: 像一般軟體一樣安裝使用
**步驟**:
```
1. 使用 Electron 包裝網頁應用
2. 建立 Windows/Mac 安裝檔
3. 分發 .exe 或 .dmg 檔案
```

### 便攜版本
**步驟**:
```
1. 建立 "便攜版" 資料夾
2. 包含所有必要檔案
3. 加入 "開始使用.txt" 說明檔
4. 壓縮後分享
```

## 🔄 更新機制

### 自動更新通知
```javascript
// 可在程式中加入版本檢查
const checkForUpdates = async () => {
    try {
        const response = await fetch('https://your-update-server.com/version');
        const latestVersion = await response.text();
        if (latestVersion !== currentVersion) {
            showUpdateNotification(latestVersion);
        }
    } catch (error) {
        console.log('無法檢查更新');
    }
};
```

### 手動更新說明
```
1. 定期檢查原始分享來源
2. 下載最新版本
3. 備份現有資料
4. 替換舊檔案
5. 匯入備份資料
```
