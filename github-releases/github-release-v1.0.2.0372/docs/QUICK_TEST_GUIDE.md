# 快速測試指南

**目的**：驗證 UI/UX 優化效果  
**預估時間**：10-15 分鐘

---

## 🚀 立即測試

### 步驟 1：啟動開發伺服器（1 分鐘）

```bash
npm run dev
```

等待伺服器啟動，然後開啟瀏覽器訪問 `http://localhost:5173`

---

### 步驟 2：視覺檢查（5 分鐘）

#### 2.1 檢查 StockRow 表格

**查看項目**：
- ✅ 現價、市值、持股數、成本價、損益、股息都**右對齊**
- ✅ 所有數值使用**等寬字體**（數字對齊整齊）
- ✅ 所有金額有**千分位逗號**（如：1,234,567）
- ✅ 負數使用**括號**表示（如：(87,683)）

**測試方法**：
1. 查看任一股票的數值欄位
2. 檢查是否右對齊
3. 檢查是否有千分位逗號
4. 如果有負數，檢查是否用括號

---

#### 2.2 檢查 PortfolioStats 卡片

**查看項目**：
- ✅ 標籤是**小字淡色**（如：總市值）
- ✅ 數值是**大字粗體**（如：1,234,567）
- ✅ 數值**右對齊**
- ✅ 總損益卡片有**背景色和邊框**（綠色或紅色）

**測試方法**：
1. 查看頂部的統計卡片
2. 檢查標籤和數值的視覺對比
3. 檢查總損益卡片是否有特殊樣式

---

### 步驟 3：功能測試（3 分鐘）

#### 3.1 測試隱私模式

**操作**：
1. 點擊右上角的眼睛圖示
2. 檢查數值是否變成 `****`
3. 再次點擊，檢查數值是否恢復

**預期結果**：✅ 隱私模式正常運作

---

#### 3.2 測試數值計算

**操作**：
1. 查看任一股票的損益
2. 手動計算：(現價 - 成本價) × 持股數
3. 檢查是否與顯示的損益一致

**預期結果**：✅ 數值計算正確

---

#### 3.3 測試編輯功能

**操作**：
1. 點擊任一股票的持股數或成本價
2. 修改數值
3. 按 Enter 保存
4. 檢查數值是否更新

**預期結果**：✅ 編輯功能正常

---

### 步驟 4：手機版測試（3 分鐘）

#### 4.1 切換到手機模式

**操作**：
1. 按 F12 開啟開發者工具
2. 點擊「響應式設計模式」圖示（或按 Ctrl+Shift+M）
3. 選擇「iPhone 12 Pro」或其他手機尺寸

---

#### 4.2 檢查手機版顯示

**查看項目**：
- ✅ 表格可以**橫向滾動**
- ✅ 數值**對齊正常**
- ✅ 卡片**顯示正常**
- ✅ 所有功能**可以操作**

**測試方法**：
1. 橫向滑動表格
2. 檢查所有欄位是否可見
3. 點擊各種按鈕測試功能

---

## 🐛 常見問題

### Q1: 數值顯示為 `--`？
**原因**：數值為 `undefined` 或 `NaN`  
**解決**：檢查數據是否正確載入

### Q2: 負數沒有括號？
**原因**：可能是正數或零  
**解決**：確認數值確實為負數

### Q3: 千分位逗號沒有顯示？
**原因**：數值可能小於 1000  
**解決**：測試大於 1000 的數值

### Q4: 手機版表格被截斷？
**原因**：表格容器沒有 `overflow-x-auto`  
**解決**：已在修改中處理，應該正常
