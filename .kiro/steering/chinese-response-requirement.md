# Chinese Response Requirement

## 🎯 核心原則

### 絕對要求
- ✅ **所有回覆都必須使用中文**: 每個回覆、解釋和溝通都必須使用中文
- ✅ **無例外**: 無論用戶使用什麼語言或之前的對話語言
- ✅ **專業中文**: 使用清晰、專業的中文，適合技術討論

## 📋 實施規則

### 回覆語言標準
- **主要語言**: 僅使用中文
- **技術術語**: 使用標準中文技術術語
- **代碼註釋**: 所有代碼註釋必須使用中文
- **文檔**: 所有文檔和解釋使用中文
- **錯誤訊息**: 所有錯誤訊息和日誌使用中文

### 例外情況（無）
- ❌ **不使用英文回覆**: 即使用戶使用英文
- ❌ **不混合語言**: 不要混合中文和英文
- ❌ **不切換語言**: 在整個對話中保持中文

## 🔧 實際應用

### 代碼和技術內容
```typescript
// ✅ 正確：中文註釋
// 此服務處理從 Vercel Edge Functions 獲取股價
class VercelStockPriceService {
  // 實作細節...
}

// ❌ 錯誤：英文註釋
// This service handles stock price fetching from Vercel Edge Functions
```

### 文檔和解釋
```markdown
✅ 正確："Vercel Edge Functions API 為雲端環境提供穩定的股價資料。"

❌ 錯誤："The Vercel Edge Functions API provides stable stock price data for cloud environments."
```

### 用戶溝通
- **用戶使用英文提問** → **用中文回覆**
- **用戶使用中文提問** → **用中文回覆**
- **混合語言查詢** → **用中文回覆**

## 💡 最佳實踐

### 專業溝通
1. **清晰簡潔**: 使用直接的中文表達
2. **技術準確性**: 在中文中保持技術精確性
3. **一致術語**: 使用一致的中文技術術語
4. **正確語法**: 遵循標準中文語法規則

### 代碼文檔
1. **函數註釋**: 全部使用中文
2. **變數命名**: 使用英文命名慣例（代碼中）
3. **錯誤處理**: 中文錯誤訊息
4. **日誌記錄**: 所有日誌訊息使用中文

- **每個回覆**: 必須使用中文
- **所有文檔**: 必須使用中文
